Get 방식 요청으로 응답한 페이지

HTTP요청 메서드 중 하나로 서버에서 정보를 가져올 때 사용되는 요청 방식이다.

특징 1. 요청시 UPL의 쿼리 문자열(Query String)을 포함하여 데이터를 전송
=> 사용자가 입력한 값(데이터)를 서버에 보낼때 URL이 노출되어 전달함.
=> 보안 유지가 어려움
=> 즉, 로그인이나 회원가입같은 private 데이터가 포함된 요청인 경우 Get방식은 부적합 함.

특징 2. 데이터 길이 제한이 있음
=> URL에 함께 전달되는 방식이기 때문에 데이터의 길이 제한이 있음, 너무 긴 데이터는 잘린채 전달될 수 있음(평균2000자 정도)
=> 즉, 게시판 작성같은 경우 Get방식이 부적합함.

특징 3. 캐싱 기능 및 즐겨찾기 사용 가능
URL을 저장해둔 뒤 재요청하는 방식
=> 검색기능에 적합
